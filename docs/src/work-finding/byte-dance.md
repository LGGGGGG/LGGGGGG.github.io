## 八股

1. 为什么还会 OOM
   超出虚内存+物理内存大小
2. 虚函数表在内存什么位置
   只读数据段或常量区
3. 虚函数可以内联，但是表现多态性时不能，只有当所调用对象是哪个类时才可以内联。
4. include cpp 会多重定义
5. 成员函数可以 delete this,被调用的方法不涉及这个对象的数据成员和虚函数就没事。
6. cpp 内存：堆(动态)、栈(局部变量)、text(代码，在可执行文件中)、bss(未初始化的全局变量和静态变量，不在可执行文件中由系统初始化)、data(已初始化的静态变量、在可执行文件中)、rodata(只读、所有常量、在可执行文件中)
7. 临时变量 emplace_back()构造不拷贝用，push_back(构造又拷贝)
8. 在 main 函数之前执行逻辑
   全局变量构造、全局变量、对象和静态变量、对象空间分配和赋初值、全局变量赋值函数、全局 lambada
   https://blog.csdn.net/qq_38836770/article/details/108871608
9. OOM
   https://blog.csdn.net/m0_38048955/article/details/135282916
10. vector<bool>不是 STL 容器，他是一个特殊的模板，存的是 bit，用 auto x = v[i]访问元素返回的是引用，会改变原值的，https://blog.csdn.net/m0_72877724/article/details/140507247
11. 函数重写只要求签名一致、返回类型可以不一致。
12. 栈上分配对象是使用对象而非指针，并且封装在函数里，把构造函数设为私有或者保护，用 delete 删掉 new,用智能指针
13. vector 线程安全 1.加锁 2.atomic3.thread_local(读有问题)
14. B 树结点存 key value，B+树只有叶子存 value
